## 1. 컴퓨터의 기본 구조

### 1.1 컴퓨터시스템의 구성
컴퓨터시스템은 아래와 같이 구성되어 있다.
![](https://velog.velcdn.com/images/ukja2/post/a5d65354-54ea-404e-ae62-7799104095e7/image.png)

#### 하드웨어(hardware)
- 컴퓨터 정보들의 전송 통로를 제공해 주고, 그 정보에 대한 처리가 `실제로 일어나게 해주는 물리적인 실체`를 의미한다.

#### 소프트웨어(software)
- 정보들이 이동하는 방향과 정보 처리의 종류를 지정해주고, 그러한 동작들이 일어나는 시간을 지정해주는 명령(command)들의 집합을 의미한다.

- 소프트웨어는 하드웨어에 통해 실행되고, 동작하기 때문에 매우 의존적이다.

- `시스템 소프트웨어` (System software): 컴퓨터 시스템의 구동을 위해 필요한 소프트웨어이다. 
Ex)OS(Windows, Unix, Linux 등), 각종 유틸리티 프로그램들(프린터 드라이버 등)

- `응용 소프트웨어` (application software): 사용자의 구체적인 업무에 도움을 주는 소프트웨어이다.
Ex)워드프로세서, 웹브라우저, MS-Excel 등



#### 1.1-1 컴퓨터의 기능
컴퓨터는 프로그램 코드들을 정해진 순서대로 실행하는데, 그 과정에서 필요한 데이터를 `읽고(read)`, `처리(Processing)`하고, 그 결과를 `저장(store)`한다.

이러한 기능들을 수행하는 컴퓨터는 아래와 같이 여러 요소들이 상호 연결되어 구성된다.

#### 1.1-2 주요 구성요소들: 중앙처리장치, 기억장치, 입출력장치

![](https://velog.velcdn.com/images/ukja2/post/ef0e7500-65d3-46b5-b824-78d4f185aab9/image.png)

- 중앙처리장치(Central Processing Unit, CPU): CPU는 `프로세서` 라고 불리기도 하며, `프로그램 실행` 과 `데이터 처리` 라는 중추적인 기능의 수행을 담당하는 요소이다.

- 기억장치: CPU가 처리할 프로그램 코드와 데이터가 저장되는 곳으로 이는 특성과 속도 및 용도에 따라 `주기억장치`와 `보조기억` 장치로 나뉜다.

1. 주기억장치(main memory)
	- CPU 가까이에 위치하며, 반도체 기억장치 칩들로 구성
    - 고속의 읽기 및 쓰기 가능
    - 가격이 높고 면적을 많이 차지 -> 저장 용량의 한계
    - 영구 저장 능력의 부재로 프로그램 실행 중에만 일시적으로 사용되는 기억장치이다.

2. 보조저장장치(auxiliary storage device)
	- 2차 기억장치(secondary memory)
    - 저장 밀도가 높고, 비트 당 가격이 낮음
    - 읽기/ 쓰기 속도가 느리다.
    - 영구 저장 능력을 가진 저장장치이다. EX)하드디스크, 플래시메모리, SSD, CD-ROM 등
    
3. 입출력장치(I/O device)
	- 입력장치와 출력장치의 통칭
    - 사용자- 컴퓨터 간의 상호작용을 위한 장치들
    - CPU는 해당 장치 제어기(device controller)를 통하여 동작 제어
    - `주변장치`라고도 부름

### 1.2 정보의 표현과 저장
- 컴퓨터 정보란 `2진수 비트`들로 표현된 `프로그램 코드`와 `데이터`를 의미한다.


- `프로그램 코드`는 컴퓨터가 어떤 작업을 수행할지 정의하는 명령어의 모음이다. CPU는 이 명령어들을 실행하여 작업을 수행한다.

- `데이터`는 프로그램이 작업을 수행할 때 처리해야 하는 정보이다. 프로그램 코드는 데이터를 기반으로 계산을 하거나 변환 등의 작업을 수행한다.



>#### 1. 기계어(machine language)
    - 기계 코드라고도 한다.
    - 컴퓨터 하드웨어 부품들이 이해할 수 있는 언어로서, 2진 비트들로 구성
    
>#### 2. 어셈블리어(assembly language)
    ◽ 고급 언어와 기계어 사이의 언어
    ◽ 기계어가 CPU 내부 구조마다 하드웨어가 이해할 수 있는 언어가 서로 다르기 때문에 
    이러한 언어상의 차이를 해결하기 위해 나타난 `고급 언어`와 `기계어` 사이의 언어
    ◽ 어셈블러로 번역되며, 기계어와 1:1대응
   
>#### 3. 고급 언어(high-level language)
    ◽ 영문자와 숫자로 구성되어 인간 친화적 언어
    ◽ c, c++, python, java 등
    ◽ `컴파일러`를 이용하여 기계어로 번역
    


#### 1.2-1 프로그램 언어 번역과정
![](https://velog.velcdn.com/images/ukja2/post/8601ece8-140e-4821-ac27-e819966da97b/image.png)


해당 예시에서 고급 언어 프로그램은 기억장치에 저장되어 있는 X와 Y를 더하고, 결과겂을 기억장치의 Z번지에 저장하라는 것인데 이를 `어셈블리 프로그램으로 번역하면 아래와 같아진다.

LOAD A,X: 기억장치 X번지의 내용을 읽어서 레지스터 A에 적재(load)하라.
ADD A,Y: 기억장치 Y번지의 내용을 읽어서 레지스터 A에 적재된 값과 더하고, 결과를 레지스터 A에 적재하라.
STOR Z,A: 레지스터 A의 내용을 기억장치 Z번지에 저장(store)하라.

- 프로그램 언어 번역 소프트웨어
	◽ 컴파일러: 고급언어 프로그램을 기계어 프로그램으로 번역하는 소프트웨어
    ◽ 어셈블러: 어셈블리 프로그램을 기계어 프로그램으로 번역하는 소프트웨어
    ◽ `니모닉스` : 어셈블리 명령어가 지정하는 연산을 가리키는 알파벳 기호 EX)LOAD, ADD, STOR 등

#### 1.2-2 기계 명령어의 형식
-  `연산 코드(op code)`
▪ CPU가 수행할 연산을 지정해 주는 비트들
▪ 비트 수 = ‘3’이라면, 지정될 수 있는 연산의 최대 수: 2^3 = 8개

- `오퍼랜드(operand)`
▪ 연산에 사용될 데이터 혹은 그것이 저장되어 있는 기억장치 주소(memory address)
▪ 비트 수 = ‘5’ 라면, 주소지정(addressing) 할 수 있는 기억 장소의 최대 수: 2^5 = 32 개

#### 1.2-3 프로그램 코드(명령어)와 데이터는 지정된 기억 장소에 저장
- `단어(word)` 단위로 저장
▪ **단어**: 각 기억 장소에 저장되는 정
보의 기본 단위로서, CPU에 의해 한 번에 처리될 수 있는 비트들의 그룹
▪ **단어 길이의 예**: 8비트, 32비트, 64비트, 128비트
▪ **주소지정 단위**: 단어 단위 혹은 바이트(byte) 단위
![](https://velog.velcdn.com/images/ukja2/post/e82820be-be52-444e-89b9-ceff2ed6a72a/image.png)

### 1.3 시스템의 구성
#### 1.3-1 CPU와 기억장치의 접속
![](https://velog.velcdn.com/images/ukja2/post/b6498685-f515-45d6-9e23-977a3c0b195f/image.png)

`시스템 버스(system bus)`
-  CPU와 시스템 내의 다른 요소들 사이에 정보를 교환하는 통로
-  기본 구성
o 주소 버스(address bus)
o 데이터 버스(data bus)
o 제어 버스(control bus)

`1. 주소 버스(address bus)`
▪ CPU가 외부로 발생하는 주소 정보를 전송하는 신호 선들의 집합
▪ 주소 선의 수는 CPU와 접속될 수 있는 최대 기억장치 용량을 결정
- 단방향성(unidirectional): 주소는 CPU로부터 기억장치 혹은 I/O 장치들로 보내지는 정보


`2. 데이터 버스(data bus)`
▪ CPU가 기억장치 혹은 I/O 장치와의 사이에 데이터를 전송하기 위
한 신호 선들의 집합
▪ 데이터 선의 수는 CPU가 한 번에 전송할 수 있는 비트 수를 결정
- 양방향성(bidirectional): 읽기와 쓰기 동작을 모두 지원

`3.제어 버스(control bus)` 
▪ CPU가 시스템 내의 각종 요소들의 동작을 제어하기 위한 신호 선들의 집합

[예시] 
o 기억장치 읽기/쓰기(Memory Read/Write) 신호
o I/O 읽기/쓰기(I/O Read/Write) 신호
o 인터럽트(Interrupt) 신호
o 버스 제어(Bus Control) 신호
- 양방향성(bidirectional): 읽기와 쓰기 동작을 모두 지원

#### 1.3-2 CPU와 I/O 장치의 접속
-  `필요한 버스 및 제어 신호`
▪ 주소 버스
▪ 데이터 버스
▪ 제어 신호 : I/O 읽기 신호, I/O 쓰기 신호

- `접속 경로` : CPU ↔ 시스템 버스 ↔ I/O 장치 제어기 ↔ I/O 장치

`I/O 장치 제어기(I/O device controller)`

- `기능` : CPU로부터 I/O 명령을 받아서, 해당 I/O 장치를 제
어하고, 데이터를 이동함으로써 명령을 수행하는 전자회
로 장치 (예: 키보드 제어기, 프린터 제어기, 등)
- `상태 레지스터`
▪ I/O 장치의 현재 상태를 나타내는 비트들을 저장한 레지스터
▪ 준비 상태(RDY) 비트, 데이터 전송확인(ACK) 비트, 등
- `데이터 레지스터`
▪ CPU와 I/O 장치 간에 이동되는 데이터를 일시적으로 저장하는 레
지스터

#### [예시] 키보드의 데이터 입력 과정
- 키보드 제어기 :
▪ 키보드의 어떤 한 키(key)를 누르면,그 키에 대응되는 ASCII 코드가 키보드 제어
기의 데이터 레지스터에 저장되고, 동시에 상태 레지스터의 In_RDY 비트가 1로 세트

- CPU :
① 키보드 제어기로부터 상태 레지스터의 내용을 읽어서 In_RDY 비트가 세트 되었는지 검사In_RDY 비트는 데이터 레지스터에 외부로부터 데이터가 적재되었는지를 표시)
② 만약 세트 되지 않았으면, 1번을 반복하며 대기. 만약 세트 되었다면, 데이터 레지스터의 내용을 읽음

`보조저장장치들(디스크, 플래시 메모리, SSD, CD-ROM 등)`도 각 장치를 위한 제어기를 통하여 키보드나 프린터와 유사한 방법으로 접속
